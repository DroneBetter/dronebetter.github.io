accel_asc

can be used to take sums/products of functions over all elements of a type of partition structure more efficiently (in exp-sqrt time) by iterating over integer partitions as 'recipes' and counting each one's multiplicity. ie. for permutations, \(n!\) is

 lambda n: smp(lambda p: choose(n,*p)//prod(map(compose(rgetitem(1),fact),rle(p)))*prod(map(lambda c: fact(c-1),p)),accel_asc(n))
=lambda n: smp(lambda p: fact(n)//prod(p)//prod(map(compose(rgetitem(1),fact),rle(p))),accel_asc(n))

it seems worthwhile to document them all, since there are a few relations, some sequences (like ) unaware that they are part of this family, and some gaps!
